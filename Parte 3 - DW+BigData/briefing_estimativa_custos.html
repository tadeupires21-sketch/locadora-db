Briefing — Estimativa de Custos: Cloud vs On-Premise

Proposta Executiva DW + Big Data · Consórcio de locadoras (escala de cidade) · Parte 3

Insumo para dimensionamento · base para fechar a Decisão 5 (onde roda a solução)

Objetivo do exercício: produzir uma comparação de custo cloud vs on-premise (mensal + amortizado) que sirva de evidência para a Proposta Executiva. O cliente tem restrição orçamentária e busca ROI rápido, então a métrica que decide é custo total de propriedade (TCO), não pico de performance. Rodar 2–3 cenários de frota.

2 · Linhas de custo a comparar

☁ Cloud (opex)
  • Ingestão: Kafka gerenciado vs self-host em VM
  • Storage em camadas: GB/mês hot/warm/cold
  • Compute: streaming 24/7 vs batch spot
  • Egress: dado que sai da cloud (costuma surpreender)
  • NoSQL gerenciado: Atlas / Astra / Redis Cloud
  • DW gerenciado: Postgres RDS/Cloud SQL
🖥 On-premise (capex + opex)
  • Hardware: servidores/storage (amortizar em 3–5 anos)
  • Energia + rack + refrigeração
  • Pessoal de operação (o maior custo escondido)
  • Redundância/backup para tolerância a falha
  • Sincronização cloud↔on-prem (se híbrido)
  • Licenças — evitáveis com stack open-source

3 · O que entregar de volta

  1. Tabela cloud vs on-prem com custo mensal por linha, para os 2–3 cenários de frota.
  2. On-prem amortizado (capex diluído em 36–60 meses) para comparar maçã com maçã contra o opex da cloud.
  3. Provedor de referência usado (AWS/GCP/Azure) e se o preço é self-host em VM ou serviço gerenciado.
  4. Uma linha de recomendação: qual ganha em TCO sob a premissa de orçamento apertado + ROI rápido.
Premissas a confirmar antes de rodar: (a) tamanho de frota dos cenários; (b) frequência e tamanho dos clipes de emergência (item mais sensível); (c) janela de retenção do cold storage (1, 3, 5 anos?); (d) se a comparação on-prem pressupõe um data center já existente ou compra do zero. Ajuste qualquer um e re-rode.

Contexto: a inclinação inicial era híbrido, mas estava ancorada num requisito de sigilo que veio do escopo genérico, não do enunciado real — que torna o on-prem opcional. Sob a premissa de custo, single-cloud é o candidato mais forte; esta estimativa serve para confirmar (ou refutar) com número. Decisão 5 fica aberta até os resultados.
